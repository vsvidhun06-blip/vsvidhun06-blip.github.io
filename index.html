Interactive Exploration of Concurrent Programs' Executions

Java 21 / JavaFX desktop tool that lets developers step through concurrent program executions interactively — visualising memory operations as live graph nodes, with happens-before and synchronises-with edges updating in real time as each execution choice is made. Closes a gap left by tools like herd7 and CDSChecker, guiding step-by-step construction of each execution rather than enumerating them all at once — making it significantly easier to reason about why outcomes are allowed or forbidden on ARM and POWER.

International Conference on Concurrency Theory · Tool Paper

An Interactive Visualiser for Weak Memory Executions under the WEAKEST Model

V. Suja, M. Doko (Heriot-Watt University)

We present a pedagogical tool for exploring executions of concurrent programs under the WEAKEST weak memory model. The system renders event structures as interactive graphs, visualises conflicting branches via DFS-based enumeration, and contrasts Sequential Consistency with WEAKEST semantics on a curated litmus test library (SB, LB, MP, CYC, IRIW). A 9-participant user study evaluates comprehension gains over static presentation.
